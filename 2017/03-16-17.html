{code} Community

A few months ago FittedCloud, a company focused on reducing public cloud cost, reached out to the REX-Ray development team. They had learned of REX-Ray thanks to one of their customers who wanted to be able to use containerized applications with attached storage in their public cloud environments. The customer had looked at REX-Ray and told FittedCloud that this was a great piece of technology, and they would like to see a FittedCloud integration to it.

After investigating the open source project, FittedCloud was very impressed and enthusiastically agreed to engage with the REX-Ray community and contribute a driver to the project.
FittedCloud's contributed driver became officially supported in REX-Ray v0.8 released earlier this month. The customer can now successfully use cloud storage and share data among Docker containers by using FittedCloud's software and REX-Ray's storage framework. Additionally, FittedCloud can actively market their AWS optimization offerings in container environments, enabling FittedCloud to expand into this rapidly growing market.

Read more about FittedCloud, their solutions and their work with REX-Ray here.
{code} Org.

Beginning back in November 2016, Vladimir Vivien  began work on a new volume plugin that allows Kubernetes pods to consume storage from a Dell EMC ScaleIO cluster.  The source code for this project is now part of Kubernetes, scheduled to  be released in version 1.6. 

This is a big milestone for our team as ScaleIO becomes part of the core Kubernetes code and a first class storage provider. Customers looking to pursue containerized workloads with Kubernetes can utilize ScaleIO as a native storage solution. This also applies to RedHat OpenShift, which is built on Kubernetes.

With this integration, users of Kubernetes can seamlessly declare persistent volumes and claims that are backed by ScaleIO.

Find out more on the latest blog from Vlad .
