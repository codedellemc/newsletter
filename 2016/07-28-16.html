DevHigh5 Featured Project Series Continued
 
Over the last several weeks we have introduced ScaleIO Gateway , Mongoose , Puppet-ScaleIO,   Knife-Storage,  PyVMAX and Metalnx as featured DevHigh5 projects. This week we continue with the Ceilometer-ECS and StorOps projects.

As always, we are very excited to welcome several new projects to the EMC {code} GitHub site. These projects have gone through our DevHigh5 program, which is designed to recognize developers who want to contribute to open source projects. Check out the EMC {code} Community and our wiki for more information on the DevHigh5 program. 

 

Ceilometer-ECS provides files used by OpenStack Ceilometer to monitor the usage meters of ECS instances. With this package installed and configured, the Ceilometer Central Agent will poll remote ECS instances via REST API  to efficiently collect, normalize and transform metrics produced by ECS. Jiale Huo has shared his project here.

StorOps is a minimalist Python library to manage VNX/Unity systems. With it, you can create, manage, and delete storage pools, volumes, shares and much more. Created by the EMC OpenStack team, you can find the code and documentation here.

Are you working on a project, looking for a project to focus on or have a great idea but don't know how to get started? Reach out to us - send us an email, hit us up on Twitter or connect on Slack. We look forward to working with you!

Monitoring EMC ScaleIO with Grafana using Intel SDI's Snap by Drew Smith

Drew Smith recently blogged about using the  Grafana monitoring tool as a method for storing and visualizing EMC's Elastic Cloud Storage metrics. This week he's gone further, and explored how to use Grafana to do the same thing for ScaleIO, but with a twist!

 
Drew looked at using a new and interesting telemetry framework called snap from Intel's Software Defined Infrastructure team. With this framework he was able to poll data from ScaleIO and push it to Grafana.



Drew worked together with the Intel SDI team to enhance the functionality of snap by collaborating on a ScaleIO plugin. The result of this is that anyone can now use snap to get telemetry data from a myriad of sources, including ScaleIO, and visualize it in Grafana as seen above.

Check out the blog post by Drew for more information on how it all works.
