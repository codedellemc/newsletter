{code} Org.

The {code} team had a blast at DockerCon with a ton of great conversations around container technologies and storage persistence in particular. REX-Ray was a hot topic at the conference and our container-inspired booth definitely spurred conversations and interest in what we are doing.



During the event we also announced functionality and features in the upcoming 0.9 release of REX-Ray where we are adding great features such as:
  • {code} by Dell EMC announces REX-Ray 0.9 and security enabled by default 
  • REX-Ray managed plugins are now in the Docker Store as Docker Certified for Amazon EBS/EFS, Dell EMC ScaleIO/Isilon, Google Compute Engine PD and any S3-compatible storage 
  • Polly, the open source volume scheduler, will see its controller features integrated with REX-Ray 
  • New updates and features contribute to enterprise readiness, as {code} by Dell EMC through REX-Ray and libStorage works with industry organizations to ensure long-lasting interoperability of storage in Cloud Native through a universal Container Storage Interface
You can read more about these announcements in our press release.



There were many interesting announcements at DockerCon, the largest of which was the Moby Project, and with that also the new LinuxKit. The Moby Project is essentially a way for the individual components of Docker to be decoupled, enabling others to use just the components they need instead of all of the Docker engine. Moving forward, Docker CE will be built using the components in the Moby Project, and for end users this change will be unnoticeable as the packages and their names will stay the same.

You can read more about the the announcements by reading the blog posts for the Moby Project and LinuxKit.
