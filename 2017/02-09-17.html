DevOps, Laundry and the Speed of light 

If you're only defining DevOps in terms of continuous integration, you're missing so much of its value. You may recall Amdahl's Law if you studied computer science, which refers to the effort needed to identify the maximum improvement possible by improving any particular part of a system (IE: A chip design's max improvement opportunity is the speed of light). But how does this apply to software? How does this apply to say, DevOps? Read more on how Joshua Bernstein (VP of Technology, Dell EMC) relates laundry, the speed of light and DevOps. 

Persistent storage in Docker for AWS is not a mystery

Docker for AWS tailors itself for stateless applications leaving out Elastic Block Storage (EBS) as a critical service. EBS allows databases, key-value stores, legacy applications, and more to store their data outside and persist it beyond the container lifecycle. 

Wouldn't it be cool to have this added as an available service so even your persistent applications can be supported?

Learn more from Kenny Coleman on how to make this possible.
